Chapter 2-1

探索・推論

学習目標

この章では以下の内容を学習します：

  • 探索・推論の具体例を説明できる
  • 探索・推論を実現する代表的な手法を説明できる

キーワード

α-β法, Mini-Max法, SHRDLU, STRIPS, 探索木, ハノイの塔, 幅優先探索, 深さ優先探索, ブルートフォース, モンテカルロ法

探索・推論とは

探索・推論は、AI研究の初期から重要な分野で、問題解決のための体系的なアプローチを提供します。

探索の定義

  • 問題空間: 可能な状態とその間の関係を表現
  • 状態: 問題の特定時点での状況
  • 操作（行動）: 状態間の遷移を引き起こす動作
  • 目標: 到達したい最終状態

推論の定義

  • 演繹推論: 一般原理から特定の結論を導出
  • 帰納推論: 特定の事例から一般原理を導出
  • 類推推論: 類似した状況からの推論

探索アルゴリズム

探索木

探索木（Search Tree）は、問題の状態空間を木構造で表現したものです。

探索木の構成要素

  • 根ノード: 初期状態を表す
  • 子ノード: 操作により到達可能な状態
  • 葉ノード: 目標状態または行き止まり
  • 枝: 状態間の遷移（操作）

探索木の例：ハノイの塔

ハノイの塔は探索アルゴリズムの代表的な例題です：

  • 状態: 各棒の円盤配置
  • 操作: 円盤を別の棒に移動
  • 制約: 大きい円盤を小さい円盤の上に置けない
  • 目標: 全円盤を目的の棒に移動

n枚の円盤のハノイの塔の最小手数：

$$\text{最小手数} = 2^n - 1$$

幅優先探索（BFS）

幅優先探索（Breadth-First Search）は、探索木を横方向に展開していく手法です。

アルゴリズム

  1. 開始状態をキューに追加
  2. キューから状態を取り出し調査
  3. 目標状態なら終了、そうでなければ子状態をキューに追加
  4. 2-3を繰り返す

特徴

  • 完全性: 解が存在すれば必ず発見
  • 最適性: 最短経路を発見（コスト一様時）
  • 計算量: 時間・空間ともに $O(b^d)$（$b$:分岐因子、$d$:深度）
  • メモリ使用量: 大量のメモリが必要

深さ優先探索（DFS）

深さ優先探索（Depth-First Search）は、探索木を縦方向に掘り下げていく手法です。

アルゴリズム

  1. 開始状態をスタックに追加
  2. スタックから状態を取り出し調査
  3. 目標状態なら終了、そうでなければ子状態をスタックに追加
  4. 2-3を繰り返す

特徴

  • メモリ効率: 空間計算量 $O(bm)$（$m$:最大深度）
  • 不完全性: 無限ループの可能性
  • 非最適性: 最初に見つけた解が最適とは限らない
  • 実装簡単: 再帰的実装が容易

ブルートフォース

ブルートフォース（Brute Force）は、全ての可能性を総当たりで調べる手法です。

特徴

  • 確実性: 解が存在すれば必ず発見
  • 単純性: アルゴリズムが理解しやすい
  • 非効率性: 計算量が指数的に増加
  • 適用例: パスワードクラック、チェス全手読み

ゲーム理論と探索

Mini-Max法

Mini-Max法は、2人零和ゲームで最適戦略を見つける探索アルゴリズムです。

基本概念

  • Max層: 自分の手番、評価値を最大化
  • Mini層: 相手の手番、評価値を最小化
  • 葉ノード: ゲーム終了状態の評価値
  • バックアップ: 葉から根への評価値伝播

アルゴリズム手順

  1. ゲーム木を生成
  2. 葉ノードの評価値を計算
  3. Mini層では最小値、Max層では最大値を選択
  4. 根ノードまでバックアップ

評価関数例

チェスの評価関数：

$$\text{評価値} = \sum_{i} w_i \times f_i(s)$$

where $w_i$は重み、$f_i(s)$は状態$s$での特徴量

α-β法

α-β法（Alpha-Beta Pruning）は、Mini-Max法を効率化した探索手法です。

剪定の概念

  • α値: Maxプレイヤーが保証できる最小値
  • β値: Miniプレイヤーが保証できる最大値
  • 剪定条件: α ≥ β の時、それ以上の探索を中止

効率性

  • 理想的ケース: 探索ノード数が $O(\sqrt{b^d})$ に削減
  • 実用性: 多くの場合、2-3倍の高速化を実現
  • 手順依存: 良い手順で探索すると効果大

モンテカルロ法

モンテカルロ法は、乱数を用いた確率的探索手法です。

基本概念

  • ランダムサンプリング: 確率的に状態を選択
  • 統計的推定: 多数試行による期待値推定
  • 収束性: 試行数増加により精度向上

応用例

  • Monte Carlo Tree Search (MCTS): AlphaGoで使用
  • シミュレーション: ゲーム終了まで乱数で実行
  • 数値積分: 複雑な積分の近似計算

MCTS選択式（UCB1）

$$\text{UCB1} = \bar{X}_i + C\sqrt{\frac{\ln n}{n_i}}$$

where $\bar{X}_i$は平均報酬、$C$は探索係数、$n$は総試行数、$n_i$は個別試行数

歴史的システム

SHRDLU

SHRDLUは、テリー・ウィノグラードが1968-1970年に開発した自然言語理解システムです。

特徴

  • ブロック世界: 単純化された3次元環境
  • 自然言語対話: 英語での指示理解・実行
  • 推論能力: 「なぜ？」「どのように？」への回答
  • プランニング: 複数ステップの行動計画

対話例

  • Human: "Pick up a big red block."
  • SHRDLU: "OK." （赤いブロックを取る）
  • Human: "Why did you do that?"
  • SHRDLU: "To clear off the top of it."

限界

  • 限定的環境: ブロック世界のみに特化
  • スケーラビリティ: 実世界への適用困難
  • 知識の硬直性: 事前プログラム知識に依存

STRIPS

STRIPS（Stanford Research Institute Problem Solver）は、自動プランニングの古典的システムです。

STRIPS表現

  • 状態: 論理式の集合で表現
  • 操作: 前提条件・削除リスト・追加リスト
  • 目標: 達成すべき論理式

STRIPSオペレータ例

操作: MOVE(x, y, z)
前提条件: ON(x, y) ∧ CLEAR(x) ∧ CLEAR(z)
削除リスト: ON(x, y), CLEAR(z)
追加リスト: ON(x, z), CLEAR(y)

STRIPS探索

  1. 目標と現状の差分を特定
  2. 差分を解決する操作を選択
  3. 操作の前提条件をサブゴールに設定
  4. 再帰的に解決

試験対策キーポイント

必須暗記ポイント

📝 探索アルゴリズム

  • 幅優先探索: 完全性・最適性あり、メモリ大量使用
  • 深さ優先探索: メモリ効率良、完全性なし
  • α-β法: Mini-Max法の効率化、剪定により高速化
  • モンテカルロ法: 確率的手法、AlphaGoで使用

🎯 計算量

  • BFS: 時間・空間ともに $O(b^d)$
  • DFS: 時間 $O(b^m)$、空間 $O(bm)$
  • α-β法: 理想的には $O(\sqrt{b^d})$
  • ハノイの塔: 最小手数 $2^n - 1$

🏛️ 歴史的システム

  • SHRDLU: ブロック世界での自然言語理解
  • STRIPS: 自動プランニング、論理式表現
  • ハノイの塔: 探索問題の代表例

よくある間違い

❌ 注意すべきポイント

  • BFS・DFSの特徴を逆に理解（完全性・最適性）
  • α-β法をMini-Max法と別アルゴリズムと誤解
  • モンテカルロ法の確率的性質を理解不足
  • SHRDLUの限界（ブロック世界限定）を軽視

まとめ

  • 探索アルゴリズムは問題解決の基本手法（BFS、DFS、α-β法）
  • Mini-Max法・α-β法はゲームAIの代表的手法
  • モンテカルロ法は確率的探索の重要手法
  • SHRDLU・STRIPSは初期AI研究の重要システム
  • 探索の完全性・最適性・計算量の理解が重要
