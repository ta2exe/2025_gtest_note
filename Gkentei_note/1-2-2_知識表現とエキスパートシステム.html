Knowledge Representation and Expert Systems

学習目標

  • 知識表現とは何か説明できる

知識表現とは

知識表現（Knowledge Representation）とは、コンピュータが知識を扱いやすいように形式化・表現する方法やその研究分野のことです。人間の知識をコンピュータが処理できる形式に変換し、推論や問題解決に活用することを目的としています。

知識表現の本質

知識表現の核心は、「世界に関する知識をコンピュータプログラムが理解し操作できる形式で表現する」ことにあります。これによって、AIシステムが知識に基づいた推論や判断を行うことが可能になります。

知識表現の特徴と要件

  • 表現力：様々な種類の知識を表現できること
  • 推論可能性：表現された知識から新しい知識を導き出せること
  • 効率性：計算機での処理が効率的であること
  • 理解可能性：人間にとって理解・修正しやすいこと

知識表現の重要性

知識表現は以下のような理由から人工知能研究において重要な役割を果たしています：

  • 問題解決の効率化：適切な知識表現により、問題解決の効率が大幅に向上
  • 推論の基盤：論理的推論や意思決定の基礎となる
  • 知識の共有と再利用：標準的な形式で知識を表現することで、異なるシステム間での知識の共有が可能に
  • 人間の知能理解：知識表現研究は人間の知能の理解にも貢献

知識表現は単なる「データの保存方法」ではなく、「知識をどう構造化し、どう関連づけ、どのように推論に利用するか」という包括的な枠組みです。

代表的な知識表現手法

意味ネットワーク (Semantic Network)

意味ネットワークは、概念や実体をノードとし、それらの関係を有向エッジで表現するグラフ構造です。意味的関連性を視覚的に表現でき、人間の記憶構造に近いモデルと考えられています。

意味ネットワークの例
関係性 意味 例
is-a 「〜は〜の一種である」という分類関係 「犬は動物である」
has-a 「〜は〜を持つ」という所有関係 「犬は尻尾を持つ」
part-of 「〜は〜の一部である」という構成関係 「毛は尻尾の一部である」

オントロジー (Ontology)

オントロジーは、特定の領域における概念と、それらの間の関係を形式的に記述する知識表現の枠組みです。より構造化され厳密な意味ネットワークとも言えます。

オントロジーの特徴：

  • 概念の階層関係（タクソノミー）を定義
  • 概念間の関係を明示的に定義
  • 属性（プロパティ）による概念の詳細な記述
  • 推論ルールの組み込み

Cycプロジェクトは、人間の常識をオントロジーとして表現する大規模プロジェクトです。1984年に開始され、数十万の概念と数百万の事実を含む知識ベースを構築しました。

論理ベースの表現

論理（特に一階述語論理）を用いて知識を表現する方法です。形式的で厳密な推論が可能です。

論理表現の例

「すべての人間は死ぬ」「ソクラテスは人間である」という知識から「ソクラテスは死ぬ」と推論する：

∀x (Human(x) → Mortal(x))  // すべての人間xは死ぬ
Human(Socrates)            // ソクラテスは人間である
∴ Mortal(Socrates)         // よって、ソクラテスは死ぬ

フレーム/スクリプト

フレームは、オブジェクト指向的に知識を表現する方法で、特定の概念や状況に関連する属性（スロット）と値を持つ構造です。スクリプトは、一連の出来事や手順を表現するフレームの拡張です。

フレームの例: 「レストラン」

Frame: レストラン

  • 種類: 飲食店
  • 提供するもの: 食事, 飲み物
  • スタッフ: [シェフ, ウェイター, マネージャー]
  • 営業時間: [開店時間, 閉店時間]
  • 設備: [キッチン, ダイニングエリア, トイレ]
  • 行為: [予約する, 注文する, 食事する, 支払う]

セマンティックWeb (Semantic Web)

セマンティックWebは、Webページの内容を機械が理解できるように構造化・標準化する技術フレームワークです。RDF (Resource Description Framework)やOWL (Web Ontology Language)などの標準を用いて、Webコンテンツに意味的なアノテーションを付与します。

セマンティックWebの目的：

  • ウェブ上の情報を機械可読にする
  • 異なるウェブサービス間でのデータ統合を可能にする
  • より高度な情報検索と推論を実現する

エキスパートシステムとは

エキスパートシステム（Expert System）は、特定の専門分野における人間の専門家の知識と推論能力をモデル化したAIシステムです。専門家の持つ経験則やノウハウをルールとして実装し、専門家に近い判断や助言を行うことができます。

エキスパートシステムの歴史的背景

エキスパートシステムは1970年代から1980年代にかけて人工知能の主要な研究分野となりました。当時はまだ機械学習が現在ほど発達していなかったため、人間の専門家の知識を明示的にシステムに実装する手法として注目されました。

黎明期から現在に至る発展：

  • 1965年：スタンフォード大学でDENDRALプロジェクト開始（最初期のエキスパートシステム）
  • 1970年代：MYCINなどの医療診断システムが開発される
  • 1980年代：エキスパートシステムのビジネス応用が進み、一時的なブームに
  • 1990年代：「AIの冬」の一因となった知識獲得のボトルネック問題
  • 2000年代以降：機械学習との融合や特定ドメインでの継続的な利用

エキスパートシステムの特徴

  • 特定分野に特化：医療診断、鉱物探査、化学分析など特定の専門分野の問題解決に特化
  • 説明能力：判断の根拠や推論過程を説明できる透明性を持つ
  • ルールベース：「もし〜ならば〜」という形式のルールを使用
  • 知識の明示的表現：専門家の知識を明示的にコード化して表現
  • 不確実性の取り扱い：確信度や信頼度をルールに付与して不確実性を処理

エキスパートシステムのメリット・デメリット

メリット：

  • 専門知識の保存と共有が可能
  • 判断の一貫性が保たれる
  • 推論過程の説明が可能（説明可能性が高い）
  • 希少な専門知識を広く活用できる

デメリット：

  • 知識獲得のボトルネック（専門家の知識を抽出・形式化する難しさ）
  • ルール数の増加に伴う保守の困難さ
  • 想定外の問題への対応が難しい
  • 暗黙知（言語化しにくい知識）の表現が困難

エキスパートシステムは現代の機械学習手法と比べると柔軟性や学習能力では劣りますが、「説明可能性」や「専門知識の明示的な表現」という点では優れており、医療や法律などの分野で今でも活用されています。

エキスパートシステムの構造

エキスパートシステムは通常、以下の主要コンポーネントから構成されています：

エキスパートシステムの基本構造

エキスパートシステムの主要コンポーネント

知識ベース (Knowledge Base)

専門家の知識を格納するデータベースです。主に以下の形式で知識を表現します：

  • IF-THENルール：「もし条件Aが成り立つならば、結論Bである」という形式
  • 事実：対象領域に関する基本的事実や前提
  • ヒューリスティック：経験則や実用的なノウハウ

知識ベースの例（医療診断）

【事実】

  • 患者ID001は38.5度の発熱がある
  • 患者ID001は咳がある
  • 患者ID001は倦怠感がある

【ルール】

  • ルール1: IF 発熱 AND 咳 AND 倦怠感 THEN インフルエンザの可能性(0.7)
  • ルール2: IF 発熱 AND 咳 AND 喉の痛み THEN 風邪の可能性(0.8)
  • ルール3: IF インフルエンザの可能性 AND 急な発症 THEN インフルエンザと診断(0.9)

推論エンジン (Inference Engine)

知識ベースのルールと事実を用いて推論を行うコンポーネントです。主に以下の推論戦略があります：

前向き連鎖（Forward Chaining）

既知の事実からスタートし、ルールを適用して結論を導く方法

後ろ向き連鎖（Backward Chaining）

目標の結論から始め、その条件を満たす事実やルールを探索する方法

混合戦略

前向きと後ろ向きの手法を状況に応じて組み合わせる方法

説明機構 (Explanation Facility)

システムがどのように結論に達したかを説明するコンポーネントです。ユーザーが「なぜこの結論になったのか」「なぜこの質問をするのか」を理解するのに役立ちます。

説明機能はエキスパートシステムの重要な特徴で、ユーザーの信頼獲得と教育的目的に役立ちます。現代の「説明可能AI（XAI）」の先駆けともいえます。

ユーザーインターフェース (User Interface)

ユーザーがシステムと対話するためのインターフェースです。質問の提示、情報入力、結果表示などの機能を提供します。

知識獲得機構 (Knowledge Acquisition)

専門家からの知識を収集し、知識ベースに追加・更新するためのツールやインターフェースです。知識エンジニアが専門家との対話を通じて知識を抽出し、ルール化します。

知識獲得のボトルネック

専門家の知識をコンピュータが処理可能な形式に変換する過程の難しさは「知識獲得のボトルネック」と呼ばれ、エキスパートシステム開発の主要な課題です。以下のような問題が含まれます：

  • 専門家が自分の知識を明示的に説明できない（暗黙知の問題）
  • 知識エンジニアが専門分野を十分理解できない
  • 専門知識をルールとして形式化する難しさ
  • 大量のルールの管理と更新の困難さ

有名なエキスパートシステム

1960年代から1980年代にかけて、多くの画期的なエキスパートシステムが開発されました。以下に代表的な例を紹介します。

DENDRAL

DENDRALは、1965年にスタンフォード大学で開発された最初期のエキスパートシステムの一つです。質量分析データと核磁気共鳴から有機化合物の分子構造を推定するシステムでした。

  • 開発：Edward Feigenbaum、Joshua Lederberg、Bruce Buchanan、他
  • 目的：化学者の構造解析作業を支援
  • 特徴：特定の問題領域に焦点を当てた初めての成功例
  • 影響：後のエキスパートシステム研究に大きな影響を与えた

MYCIN

マイシン（MYCIN）は、1970年代にスタンフォード大学で開発された医療診断システムです。血液感染症の診断と抗生物質の推奨を行いました。

  • 開発：Edward Shortliffe
  • 知識ベース：約600のルール
  • 特徴：
    • 後ろ向き連鎖推論を使用
    • 不確実性を確信度因子（certainty factors）で処理
    • 「なぜ」の質問に答える説明機能
  • 成果：人間の専門家に匹敵する診断精度を達成

MYCINは実際の臨床現場で広く使用されることはありませんでしたが、医療における人工知能応用の先駆けとなり、その設計原則は多くの後続システムに影響を与えました。

ELIZA

イライザ（ELIZA）は、1966年にMITのJoseph Weizenbaumによって開発された初期の自然言語処理システムです。精神科医のカール・ロジャースの手法を模倣し、患者（ユーザー）との会話を行いました。

  • 特徴：
    • 単純なパターンマッチングと置換ルールを使用
    • ユーザーの発言を質問や言い換えに変換
    • 実際の「理解」はなかったが、対話の錯覚を生み出した
  • 影響：人間とコンピュータの自然な対話の可能性を示した
  • 教訓：システムが人間のように振る舞うだけで、ユーザーは「理解」があると錯覚する（「ELIZA効果」）

その他の重要なエキスパートシステム

歴史的に重要なその他のエキスパートシステム：

  • PROSPECTOR：鉱物探査支援システム
  • XCON/R1：DEC社のコンピュータ構成支援システム
  • INTERNIST/CADUCEUS：内科診断システム
  • HEARSAY：音声認識システム
  • PXDES：植物病害診断システム

現代の知識ベースAI

ワトソン（Watson）はIBMが開発した質問応答システムで、自然言語処理、機械学習、知識表現を組み合わせたシステムです。2011年のテレビクイズ番組「Jeopardy!」（日本の「クイズ！ヒューマン」に相当）で人間のチャンピオンに勝利したことで有名になりました。

  • 特徴：
    • 大規模な知識ベースと自然言語処理
    • 機械学習と従来の知識ベースアプローチの融合
    • 証拠に基づく回答生成と確信度評価
  • 応用分野：医療診断支援、ビジネスアナリティクス、顧客サービスなど

東ロボくん

東ロボくん（東京大学入試向けAIプロジェクト）は、国立情報学研究所（NII）などが2011年から2016年まで実施した、東京大学の入試問題を解くAIシステムの開発プロジェクトです。

  • 目標：東大入試の全科目で偏差値60以上を達成すること
  • 特徴：自然言語処理、画像認識、知識処理、推論などの技術を統合
  • 成果：センター試験模試で偏差値57.8を達成。東大二次試験相当では数学で偏差値50を超える成績

東ロボくんは従来の知識ベース手法と現代の機械学習手法を組み合わせたアプローチを採用し、特に数学や物理の問題解決には論理的推論と知識表現が重要な役割を果たしました。

現代への応用

伝統的なエキスパートシステムは現代のAIにおいてもなお重要な役割を果たしており、また新しい技術と融合することで進化し続けています。

知識表現の現代的応用

現代における知識表現の主な応用分野：

  • 質問応答システム（Question-Answering）：Googleの知識グラフ、AppleのSiri、Amazon Alexaなど
  • セマンティックWeb：リンクトデータ（Linked Data）、RDF、OWLを活用したウェブの知識構造化
  • ナレッジグラフ：実体間の関係を表現する大規模グラフデータベース
  • オントロジー工学：医学（SNOMED CT）、生物学（Gene Ontology）などの分野別知識体系

エキスパートシステムの現代的進化

現代のエキスパートシステムは、従来の手法と新しいAI技術を融合して進化しています：

  • ハイブリッドシステム：ルールベースと機械学習の組み合わせ
  • ファジーエキスパートシステム：ファジー論理を用いた不確実性の取り扱い
  • 説明可能AI（XAI）：ディープラーニングの判断を説明する機構
  • LLMとの連携：大規模言語モデルと知識ベースの組み合わせ

応用分野

知識表現とエキスパートシステムの主な現代的応用分野：

  • 医療診断支援：臨床意思決定支援システム（CDSS）
  • 金融：与信評価、不正検出、ポートフォリオ管理
  • 法律：法令遵守チェック、契約書分析
  • 製造業：異常検知、予知保全
  • カスタマーサポート：インテリジェントチャットボット
  • 教育：インテリジェント・チュータリング・システム

データマイニングとウェブマイニング

データマイニングは大量のデータから有用なパターンや知識を抽出する技術です。ウェブマイニングはWeb上のデータを対象としたデータマイニングの一種です。これらは現代の知識獲得の重要な手段となっています。

  • テキストマイニング：文書から知識やパターンを抽出
  • Webコンテンツマイニング：Webページの内容から情報抽出
  • Web構造マイニング：Webのリンク構造から関係性を分析
  • Webユーザビリティマイニング：ユーザーのWeb行動パターンを分析

これらの技術は、従来の人手による知識獲得の限界（知識獲得のボトルネック）を克服する一助となっています。

対話システムとインタビューシステム

インタビューシステムは、ユーザーとの対話を通じて情報を収集・整理するシステムです。現代では自然言語処理の進歩により、より高度な対話が可能になっています。

  • 医療問診システム：患者から症状を効率的に聞き取り
  • 顧客調査システム：ユーザー体験データの収集
  • 知識獲得インタビュー：専門家からの知識抽出を支援
  • 就職面接練習システム：模擬面接と評価を提供

知識表現とLLMの関係

最新の大規模言語モデル（LLM）は、従来の知識表現と異なるアプローチで「知識」を扱いますが、両者には相補的な関係があります：

  • LLMは統計的パターンとして暗黙的に知識を表現するのに対し、従来の知識表現は明示的で解釈可能
  • LLMは柔軟で広範囲な知識を持つが精度保証が難しく、知識表現は厳密だが範囲が限定的
  • 現代の研究では、LLMと従来の知識表現を組み合わせる「ニューロシンボリックAI」が注目されている

イライザ (ELIZA) 1966年にMITのJoseph Weizenbaumによって開発された初期の会話システム。精神科医を模倣
インタビューシステム (Interview System) ユーザーとの対話を通じて情報を収集・整理するシステム。知識獲得や問診などに利用
ウェブマイニング (Web Mining) Web上の大量のデータから有用なパターンや知識を抽出する技術
オントロジー (Ontology) 特定の領域における概念と、それらの間の関係を形式的に記述する知識表現の枠組み
セマンティックWeb (Semantic Web) Webページの内容を機械が理解できるように構造化・標準化する技術フレームワーク
データマイニング (Data Mining) 大量のデータから有用なパターンや知識を発見・抽出する技術
東ロボくん (Todai Robot Project) 国立情報学研究所が主導した、東京大学の入試問題を解くAIシステムの開発プロジェクト（2011-2016）
マイシン (MYCIN) 1970年代にStanford大学で開発された医療診断システム。血液感染症の診断と抗生物質の推奨を行った
ワトソン (Watson) IBMが開発した質問応答システム。2011年のクイズ番組「Jeopardy!」で人間のチャンピオンに勝利
知識表現 (Knowledge Representation) 人間の知識をコンピュータが処理できる形式に変換し、推論や問題解決に活用する技術や研究分野
エキスパートシステム (Expert System) 特定の専門分野における人間の専門家の知識と推論能力をモデル化した人工知能システム
知識ベース (Knowledge Base) 専門家の知識を格納するデータベース。ルール、事実、ヒューリスティックなどを含む
推論エンジン (Inference Engine) 知識ベースのルールと事実を用いて推論を行うコンポーネント。前向き連鎖や後ろ向き連鎖などの戦略を採用
知識獲得のボトルネック (Knowledge Acquisition Bottleneck) 専門家の知識をコンピュータが処理可能な形式に変換する過程の難しさを指す問題

重要ポイント

知識表現の特性と役割

知識表現は単なる「データの保存方法」ではなく、「知識をどう構造化し、どう関連づけ、どのように推論に利用するか」という包括的な枠組みである。適切な知識表現により、AIシステムの問題解決能力が大きく向上する。

エキスパートシステムの構造と特徴

エキスパートシステムは「知識ベース」「推論エンジン」「説明機構」「ユーザーインターフェース」から構成される。最大の特徴は「説明可能性」にあり、判断の根拠を明示できる点が現代のXAI（説明可能AI）の先駆けとなっている。

代表的エキスパートシステム

DENDRAL（化学構造分析）やMYCIN（医療診断）、ELIZA（対話）など、各分野で先駆的なエキスパートシステムが開発された。これらのシステムは、特定領域の専門知識をルール化し、人間の専門家に近い判断を行うことができた。

知識獲得のボトルネック

エキスパートシステムの最大の課題は「知識獲得のボトルネック」と呼ばれる、専門家の知識を形式化する困難さである。この問題はAIの発展において重要な制約となり、「AIの冬」を招いた一因ともなった。

まとめ

知識表現とエキスパートシステムは、人間の専門知識をコンピュータが扱える形に変換し活用する人工知能の重要な分野です。意味ネットワークやオントロジーなどの知識表現手法と、それを活用するMYCINやDENDRALなどのエキスパートシステムは、現代のAI発展に大きく貢献しました。これらの技術は「説明可能性」という点で優れており、現代のセマンティックウェブや質問応答システムなどにも影響を与え続けています。
